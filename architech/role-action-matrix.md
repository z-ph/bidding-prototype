# 角色×行为矩阵（需求级，状态无关）

> 本文件被 `01-requirements-analysis.md` §1.2 引用。
> 从原始需求文档逐条提取**每个动作**，映射到执行角色。状态无关——回答「这件事归谁做」。
> 状态相关的权限细化（「在什么状态下能做」）见 `04-role-state-action-matrix.md`。

## 图例

| 符号 | 含义 |
|---|---|
| ● | 执行（做这件事的主体） |
| ◐ | 审核 / 审批 / 确认 |
| ○ | 查看 / 监督 / 只读 |
| · | 无权限 |
| 自 | 仅限本人/本企业 |
| 组长 | 仅评标组长 |

角色列：招=招标人｜代=招标代理｜投=投标人｜专=评标专家｜监=监督人｜管=平台管理员｜客=游客

## 与 `04` 的关系

本矩阵是**需求级、状态无关**的责任归属，回答「谁做」。
`04-role-state-action-matrix.md` 是**设计级、状态相关**的权限细化，回答「在什么状态下能不能做」，并补充 `owner`（归属人）维度（如解密只认 BidFile.bidderId）。

落地时两层叠加：先按本矩阵确定角色可见动作，再按 `04` 在具体状态下用 `canAct()` 收敛。
