# Proposal: 页面评审意见整改（2026-07-14）

## Change ID

`page-review-20260714-fixes`

## Why

基于 `page-reviews-20260714-0031.zip` 中的 26 条页面评审意见，当前 React 原型在项目流程、角色权限、大厅入口、招标文件、评标专家、投标人工作流等多个页面存在体验不一致或功能缺失。本提案将这些可落地的评审意见转化为规范需求，集中整改。

## What Changes

1. **项目管理流程**
   - 限定只有招标人能创建项目，招标代理无创建入口（评审 #14）。
   - 项目列表增加真正的详情页、阶段化编辑控制、发标按钮（评审 #18、#19、#21）。
   - 支持邀请招标时选择/邀请投标人，并校验标段预算与项目预算关系（评审 #24、#26）。
   - 明确“需求编号”字段含义（评审 #25）。

2. **招标文件**
   - 目录支持自定义增删与拖拽排序（评审 #13）。
   - 支持一键导入模板（评审 #12）。
   - 招标人自行招标时也能填写评标办法（评审 #22）。

3. **开标大厅**
   - 开标大厅、评标大厅从顶级菜单改为项目子页面入口（评审 #11）。
   - 签到列表改为各角色状态队列 + 单独签到按钮，并可视化缺员名单（评审 #16、#17）。
   - 投标文件解密仅投标人可操作，主持人只能查看解密后文件（评审 #15）。

4. **评标大厅**
   - 评标任务改为项目列表，点击后进入详情（评审 #3）。
   - 评标组长有独立流程：统计评标结果、生成报告（评审 #2）。
   - 合并/明确“查阅资料”入口，不再作为左侧主导航项（评审 #1）。

5. **投标人工作流**
   - 投标人项目中心聚合缴费、下载招标文件、上传投标文件入口，按状态动态显示按钮（评审 #7）。
   - 项目跟踪中的“缴纳文件费”步骤动态化，未缴费时显示“去缴纳”（评审 #8）。
   - 初始报价并入上传投标文件流程；询比价项目的报价大厅在开标后启动（评审 #6）。
   - 注册/企业档案按项目资质要求类型上传资质文件，报名前检测（评审 #5、#9、#23）。

6. **通用交互**
   - 敏感操作增加二次确认弹窗（评审 #10）。
   - “主页”菜单命名与工作台页面标题统一（评审 #4）。

## Impact

- **权限调整**：`permissions.js` 移除代理创建项目权限；投标人独立菜单项合并。
- **新增页面/组件**：项目详情页、招标邀请弹窗、模板导入、目录编辑、评标任务列表、子账号等。
- **修改页面**：项目列表、项目创建、项目跟踪、开标大厅、评标大厅、投标人项目中心、企业档案、招标文件页、Layout 菜单。
- **删除/合并**：顶级菜单中的“开标大厅”“评标大厅”“缴纳费用”“下载文件”“上传投标文件”“查阅资料”等独立入口改为项目/流程内入口。

## Out of Scope

- 真实 CA 证书/UKey 驱动集成。
- 短信/邮件等真实消息通道。
- 后台管理员的页面配置功能。
