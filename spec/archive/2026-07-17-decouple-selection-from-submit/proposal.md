# Proposal: 解耦目标选择与评审提交，支持多选

## Change ID

`decouple-selection-from-submit`

## Why

当前元素模式下点击元素立即弹出评审表单，存在两个问题：

1. “选中目标”与“填写评审”两个动作耦合，用户误触页面元素就会打断当前操作。
2. 无法对一条评审关联多个目标（例如同一问题的多个相关元素），原生 ReviewTool 支持的多目标能力在自封装组件中丢失。

## What Changes

1. 元素模式：
   - 普通点击 = 选中该元素（替换当前选择），仅显示持久高亮，不弹表单。
   - Cmd/Ctrl + 点击 = 将该元素加入/移出多选集合。
2. 框定视图模式：拖拽生成的选区保留为持久目标（可连续框多个），不再自动弹表单。
3. 工具条新增“已选 N 个目标”计数、“提交评审”按钮（N=0 时禁用）与“取消选择”按钮。
4. 点击“提交评审”才弹出表单；提交后生成一条包含全部目标的评审记录（逐目标截图），并清空选择。
5. 高亮框跟随滚动重新定位；退出评审时清空选择。

## Impact

- **代码**：`src/components/PageReview/index.jsx`、`PageReview.css`（选中态样式）。
- **行为**：点击不再弹表单；评审提交入口统一为工具条“提交评审”按钮。
- **数据**：记录格式与 SDK 兼容（`targets` 数组、逐目标截图），历史记录不受影响。

## 验证

- 点击元素只高亮不弹表单；Cmd/Ctrl+点击可多选/取消。
- 连续框选多个区域均保留。
- “提交评审”生成一条含多目标与多截图的记录；列表可见。
- ESC / 退出评审清空选择并关闭。
