# Proposal: 项目创建与列表评审问题整改

## Change ID

`fix-project-creation-review-20260717`

## Why

cxy 7-16 复测与 7-14/7-15 评审导出指出项目创建/列表仍有问题：项目级「采购方式」与标段级冗余（cxy-016）、「单一来源」残留在公告/字典/帮助（cxy-017）、标段缺「报名是否需要审核」配置（cxy-015）、委托合同确认时机错误（1415-011）、代理机构只能直接指定无筛选/随机抽取（cxy-018，待确认）、创建的项目不进列表且草稿不落库（2052-004）。

## What Changes

1. **移除项目级采购方式**：采购方式仅保留在标段级；项目列表/详情的采购方式由标段派生展示（cxy-016）。
2. **清理「单一来源」残留**：NoticePublish、AdminDictionary、Help 统一为「邀请询比价」（cxy-017）。
3. **标段报名审核开关**：标段设置新增 `needRegisterAudit`（报名是否需要审核），随项目持久化，供报名审核链路消费（cxy-015）。
4. **委托合同时机**：委托合同确认从第 0 步移到「供应商要求」步骤完成之后（1415-011）。
5. **代理机构抽选**：委托代理支持按条件（资质/地区/黑名单/回避）筛选候选机构 + 随机抽取模式，抽取结果进入委托合同；保留直接指定并标注为「手动指定」（cxy-018：评审引述需求文档要求筛选+随机抽取，产品确认前按此实现）。
6. **项目列表接入 store**：ProjectList 改为读取 projectStore（mock 基线 + 新建/草稿合并），新建项目状态「待审核」、草稿「草稿」出现在列表，刷新保留；ProjectCreate 草稿保存落库（2052-004）。

## Impact

- **修改文件**：`src/views/ProjectCreate.jsx`、`src/views/ProjectList.jsx`、`src/views/NoticePublish.jsx`、`src/views/AdminDictionary.jsx`、`src/views/Help.jsx`、`src/data/projects.js`。
- **下游契约**：标段对象新增 `needRegisterAudit: boolean`（报名审核链路按此字段决定是否需人工审核）；项目对象保持 `orgMode`（自行招标/委托代理）供 TenderDoc 权限使用。
