# Proposal: 补齐项目创建与标段配置的结构性字段

## Change ID

`fix-project-creation-gaps-20260715`

## Why

2026-07-14 多轮页面评审（cxy、xyy）指出项目创建流程中标段字段不完整、缺少采购需求入口、组织方式未明确、采购方式文案错误。这些问题导致后续开标、报价、公告发布等环节数据不闭环，必须补齐。

## What Changes

### 高优先级

1. **标段字段补全**
   - 每个标段独立配置：采购方式、标书费、保证金、投标开始时间、投标截止时间。
   - 不同标段采购方式可不同。
   - 标段预算合计仍须 ≤ 项目预算。

2. **空白标段禁止提交**
   - 标段名称、编号、预算、内容、采购方式、投标起止时间为必填。
   - 任一标段不完整时禁止进入下一步/提交审核。

### 中优先级

3. **组织方式选择**
   - 项目创建第一步增加「组织方式」：自行招标 / 委托代理。
   - 自行招标：项目成员仅招标方人员。
   - 委托代理：需选择代理机构，代理机构确认委托合同后才能加入项目成员。

4. **采购需求独立入口**
   - 新增「采购需求」模块：列表、草稿、已发布/已审核状态。
   - 采购需求字段：需求类型、需求标题、预算金额、具体需求内容。
   - 创建项目时可关联已发布的采购需求，自动带入预算与内容摘要。

5. **采购方式文案修正**
   - 将「单一来源」修正为「邀请询比价」。

## Impact

- **修改文件**：`src/views/ProjectCreate.jsx`、`src/views/ProjectList.jsx`、`src/views/ProjectDetail.jsx`、`src/data/projects.js`、`src/config/permissions.js`、`src/components/Layout.jsx`、`src/router/index.jsx`。
- **新增页面/组件**：`src/views/ProcurementRequirementList.jsx`（采购需求列表）、`src/views/ProcurementRequirementEdit.jsx`（采购需求编辑）。
- **数据模型扩展**：项目/标段需增加组织方式、采购需求关联、标段费用与时间字段。

## Out of Scope

- 真实合同签署流程（保持模拟）。
- 与外部代理机构系统对接。
- 采购需求的复杂审批流（仅保留草稿/已发布/已审核三态）。
