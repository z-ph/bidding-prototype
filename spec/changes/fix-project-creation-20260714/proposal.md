# Proposal: 修复项目创建与标段配置的结构性缺失

## Change ID

`fix-project-creation-20260714`

## Why

2026-07-14 23:11 页面评审发现项目创建流程存在多处与需求文档不符或字段缺失的问题，会导致后续开标、报价、公告发布等环节数据不闭环。

## What Changes

### 高优先级

1. **采购需求独立入口**
   - 按需求文档增加「采购需求」模块：需求发布 → 类型管理 → 需求草稿 → 需求已发布/已审核。
   - 采购需求字段：需求类型、需求标题、预算金额、具体需求内容。
   - 创建项目时可关联已发布的采购需求。

### 中优先级

2. **标段字段补全**
   - 每个标段独立配置：采购方式、标书费、保证金、投标开始时间、投标截止时间。
   - 不同标段采购方式可不同。

3. **组织方式选择**
   - 项目创建第一步增加「组织方式」：自行招标 / 委托代理。
   - 自行招标：项目成员仅招标方人员。
   - 委托代理：需选择代理机构，代理机构确认委托合同后才能加入项目成员。

4. **采购方式文案修正**
   - 将「单一来源」修正为「邀请询比价」（若原意是邀请询比价）。

## Impact

- **修改文件**：`src/views/ProjectCreate.jsx`、`src/data/projects.js`（或新增 store）、`src/config/permissions.js`（如新增菜单）。
- **新增页面/组件**：采购需求列表、采购需求编辑、需求类型管理。
- **数据模型扩展**：项目/标段需增加组织方式、采购需求关联、标段费用与时间字段。

## Out of Scope

- 真实合同签署流程（保持模拟）。
- 与外部代理机构系统对接。

## 依赖

- 需先确认需求文档中采购需求的完整字段与状态流转。
