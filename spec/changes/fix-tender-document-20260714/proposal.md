# Proposal: 修复招标文件结构化与投标人互动能力

## Change ID

`fix-tender-document-20260714`

## Why

2026-07-14 23:11 页面评审发现当前招标文件以富文本为主，缺少结构化字段；投标人无法对招标文件提出正式质疑，复核人字段也缺少需求依据。

## What Changes

1. **招标文件结构化字段**
   - 招标公告部分从富文本中拆分为结构化字段：项目名称、项目编号、开标时间、截止时间、评标方法、开标一览表、报价字段配置。
   - 这些字段在项目创建/标段配置中维护，招标文件页面只读引用或按模板渲染。

2. **投标人质疑招标文件入口**
   - 在投标人「项目中心 / 招标文件下载」页面增加「质疑招标文件」按钮。
   - 点击后弹出表单：质疑类型（商务/技术/其他）、质疑内容、附件上传。
   - 质疑提交后招标人/代理可在「异议管理」中查看与回复。

3. **复核人字段处理**
   - 若需求文档未定义「复核人」，则从 TenderDoc 页面移除该字段。
   - 若需求定义了复核流程，则按需求实现复核人选择与复核记录。

## Impact

- **修改文件**：`src/views/TenderDoc.jsx`、`src/views/BidDownload.jsx`（或投标人项目中心）、`src/views/ObjectionManage.jsx`。
- **新增页面/组件**：招标文件质疑弹窗、质疑列表。
- **数据模型扩展**：项目/标段增加结构化招标公告字段；异议/质疑对象增加 `tenderDocQuestion` 类型。

## Out of Scope

- 真实文件签章（保持模拟）。
- 与外部造价/审计系统对接。

## 依赖

- 需先完成 `fix-project-creation-20260714` 中的标段字段补全与采购方式修正，以便招标文件能引用结构化数据。
