# Proposal: 修复 2026-07-14 页面评审暴露的 P0/P1 问题

## Change ID

`fix-review-issues-20260714`

## Why

`2026-07-14-yy0-前端页面评审结果.md` 在原 zip 评审基础上进一步验证，发现前序 agent 实现存在业务规则未闭环、角色边界未隔离、前后半段数据未衔接等问题。其中 3 项 P0 必须立即修复，9 项 P1 需要在本次迭代内补齐。

## What Changes

### P0

1. **开标大厅解密权限隔离**
   - 仅投标人能解密本企业投标文件。
   - 招标人、代理、监督只能查看状态或推进流程。

2. **投标文件上传签章加密闭环**
   - 上传页提供上传 → 签章 → 加密 → 提交的完整动作链。
   - 签章/加密状态可见，支持重新加密。

3. **投标文件加密规则一致性**
   - 正式提交通道只接受 CA 证书加密。
   - 密码加密明确标记为草稿/演示用途，不进入正式提交结果。

### P1

4. **投标文件与招标文件评审条款关联**
   - 投标工具界面列出招标文件评审条款。
   - 支持把投标文件内容逐项挂接到条款。

5. **报价字段继承项目/标段配置**
   - 创建项目时可配置报价字段模板。
   - 在线报价页按模板渲染字段。

6. **评标办法驱动专家评分项**
   - 招标文件评标办法配置评分项/权重。
   - 专家评分页与汇总页按配置展示，不再固定商务30/技术40/价格30。

7. **文件版本统一链**
   - 招标文件附件、下载页、专家查阅页引用同一文件对象/版本链。
   - 变更后明确提示当前有效版本。

8. **项目跟踪按钮角色过滤**
   - 招标人视角不显示“上传投标文件”等投标人动作按钮。

9. **空白标段禁止提交**
   - 创建项目时每个标段必须完整填写名称、编号、预算、内容。

10. **开标大厅文案与签到表一致性**
    - 步骤描述与签到表对象保持一致；专家若参与签到则列表体现，否则文案移除专家。

11. **专家任务链路**
    - 增加专家抽取结果、任务授权/通知、专家接收任务页面或流程。
    - 专家明确知道因哪个项目、哪次抽取、何时收到任务、去哪里评标。

## Impact

- **修改页面**：`OpeningHall.jsx`、`BidUpload.jsx`、`ProjectCreate.jsx`、`ProjectTrack.jsx`、`TenderDoc.jsx`、`ExpertProject.jsx`、新增专家任务页面。
- **新增页面/组件**：投标文件条款关联面板、专家任务中心、文件版本组件。
- **数据模型扩展**：`portalStore.js` 或新增 store 需支持文件版本链、报价模板、评标办法配置。
- **删除/调整**：密码加密不再作为正式提交结果。

- 真实短信/邮件通知通道（保持模拟）。
- 与外部专家库/工商系统对接。
