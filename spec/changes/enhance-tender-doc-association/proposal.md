# Proposal: 增强招标文件与投标文件的关联能力

## Change ID

`enhance-tender-doc-association`

## Why

当前平台的招标文件编制（`TenderDoc.vue`）与投标文件上传（`BidUpload.vue`）之间缺少业务关联：

1. **招标文件未关联项目**：`TenderDoc.vue` 直接编辑通用章节，没有选择具体招标项目，生成的招标文件无法对应到实际项目。
2. **投标文件未关联招标要求**：`BidUpload.vue` 中项目和标段为硬编码，文件类型与招标文件要求的“商务标/技术标/报价标”结构脱节，无法校验投标文件完整性。
3. **缺少投标文件编制引导**：投标人下载招标文件后，没有页面按招标文件模板生成投标文件清单，容易漏传必要文件。

增强关联能力后，可让招标流程从“编制 → 发布 → 下载 → 编制投标 → 上传”形成闭环。

## What Changes

1. 在 `TenderDoc.vue` 顶部增加项目选择器，招标文件内容按所选项目的标段、采购方式等字段动态展示。
2. “生成招标文件”前校验是否已选择项目，生成后将文件版本与项目绑定。
3. 在 `BidUpload.vue` 中，选择项目后自动拉取该项目的标段列表和招标文件要求的文件类型清单。
4. 上传文件时要求每份文件对应一个“文件类型”（商务标/技术标/报价标/其他），并校验是否齐全。
5. 在投标人工作台或 `BidUpload.vue` 中增加“投标文件制作向导”，按招标文件模板列出必填文件清单。

## Impact

- **代码**：`src/views/TenderDoc.vue`、`src/views/BidUpload.vue`、`src/views/BidDownload.vue`（可选引导入口）
- **数据**：需要项目与标段、文件类型模板的关联数据（当前用 mock 数据扩展）
- **用户交互**：招标代理编制文件前必须先选项目；投标人上传时按模板逐项上传，降低废标风险。
- **无外部依赖变更**：仅前端视图与 mock 数据调整。
