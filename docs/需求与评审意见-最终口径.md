本文件是需求与评审意见的**唯一入口和效力仲裁**，持续维护。逐条明细不堆在这里，分别维护在两个台账：需求台账 = `docs/20260717-需求确认整理.md`，评审意见台账 = `src/views/ReviewChangeList.jsx`（页面 `/admin/review-change-list`）。

## 0. 维护规则（先读）

1. **时间戳**：提取出的每条需求/意见必须带来源日期；批次 id 含日期前缀（如 `0717-rm-004`、`cxy-013`）。
2. **新口径优先**：新需求与旧需求矛盾时，以日期较新者为准。旧条目不删除，标记「已被 YYYY-MM-DD 口径作废」并指向新条目。评审意见同理——在 `ReviewChangeList.jsx` 中将旧意见 `status` 标为「无需修复」，`fix` 引用作废它的新口径清单号（已有范例：`cxy-013` 被 0717 清单 26 作废）。
3. **不维护单批评审的逐条验证 checklist**：这类清单绑定具体批次、需求一变就过期（原 `docs/page-review-20260714-checklist.md` 已因此删除）。复核依据 = 本文件口径 + 台账状态 + 设计原则 skill `.agents/skills/workflow-first-ux-design/SKILL.md` + `AGENTS.md` 红线清单。
4. 每次需求确认会议或评审批次后，必须更新本文件的对应时间线；需求台账换代时（如 0717 之后又有新确认），新台账文件入库、旧台账效力降级为历史，均在下表登记。

## 2. 当前有效需求口径摘要（2026-07-17）

全文口径见 `docs/20260717-需求确认整理.md` §1（按主题 17 类）；与旧原型/提案的冲突处置见其 §2（14 项）。最关键的新口径覆盖点（旧需求相应作废）：

- 无每次招标的报名与资格审核；供应商一次性入库（清单 10/11/12，概要二）→ 旧报名流、按项目准入作废
- 不实现支付；保证金/文件费不缴纳；采购结果出来后由中标人缴纳投标费用，收款方待确认（清单 26，概要七）→ 在线缴费、标书费/保证金字段作废
- 无合同归档环节（清单 33，概要五）；无供应商异议环节（清单 44/45）→ ContractArchive、ObjectionManage 类功能作废
- 代理机构直接指定、委托合同不审核（清单 71/72，概要六）→ 随机抽取、委托审批作废
- 评标条款由系统识别关联，未关联允许提交（清单 36）→ 手工关联方向作废
- CA 一期不真实接入、仅预留接口；一期账号密码/手机号登录（清单 2/4/70，概要一）→ 按真实 CA 设计的加解密作废
- 短信/邮件仅预留接口；审批通知用站内信（概要四，清单 54）
- 邀请询比价不开标不评标，直接进采购结果（清单 20）→ 统一开标评标流程按采购方式分流
- 中标结果需审批但不在本系统完成（清单 31）→ 系统内只记录审批结果

云文档内部口径张力（报名 vs 投标登记、费用台账范围、合同公示、入库性质、短信程度、CA 一期边界）共 6 项仍待甲方确认，见台账 §3；清单 E 列未结追问见台账 §4。
